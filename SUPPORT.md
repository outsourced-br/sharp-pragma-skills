# Support

Use GitHub issues for bug reports, documentation problems, skill behavior anomalies, and release packaging problems.

Use discussions, if enabled, for usage questions and broader design proposals.

For security-sensitive reports, follow `SECURITY.md` instead of posting details publicly.

Before opening an issue, include:

- the package version from `VERSION`,
- the skill name,
- the exact prompt or workflow that triggered the problem,
- the expected behavior,
- the actual behavior,
- any relevant repository shape, target framework, or tool context.
