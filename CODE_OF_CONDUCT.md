# Code of Conduct

## Standard

Participation in this project must stay professional, constructive, and focused on improving the skill pack.

Expected behavior:

- Be direct about technical problems without personal attacks.
- Assume good faith unless there is clear evidence otherwise.
- Keep criticism tied to files, behavior, documentation, or tests.
- Respect maintainers' decisions on scope, licensing, and release timing.

Unacceptable behavior:

- Harassment, threats, discriminatory language, or sexualized content.
- Publishing private information without consent.
- Repeatedly derailing issues or pull requests after a maintainer decision.
- Submitting security details in public when a private disclosure path is available.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject contributions; or block participants who violate this standard. Enforcement decisions should be proportionate to the behavior and documented when practical.
